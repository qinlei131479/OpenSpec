# 文档上传与识别及大模型处理设计说明书

## 1. 背景与目标
本项目旨在为 `ArchSpec` 系统增加通用的文档上传与识别基础功能，并深度集成大模型（LLM）能力。核心目标是**支持用户上传项目相关的非结构化文档（PDF、Word、图片等），自动识别并提取项目概况信息（如项目名称、用地性质、建筑面积等），其提取结果应与用户直接输入文字识别的结果保持一致**。

技术路线采用阿里云[文档解析（大模型版）](https://help.aliyun.com/zh/document-mind/developer-reference/document-parsing-large-model-version)将文档转换为结构化的 Markdown，进而通过 Agent 复用统一的 Prompt 进行语义理解和信息提取。

## 2. 总体架构设计

本功能涉及前端 (`apps/web`)、后端 (`apps/backend`) 和 AI 代理 (`apps/agent`) 三个模块的协作。

*   **Frontend (Web)**:
    *   在 `Editor.vue` 的“项目概况”区域提供文件上传入口。
    *   支持拖拽上传图片、PDF、Word 文件。
    *   接收解析后的 JSON 数据并自动填充表单。
*   **Backend (Java)**:
    *   接收文件上传，负责文件的临时存储或上传至 OSS。
    *   作为中转层，调用 Agent 的解析接口，透传文件路径或 URL。
*   **Agent (Python)**:
    *   **统一解析引擎**：集成 `DocParser` (基于阿里云 DocMind) 将各类文档统一转换为 Markdown。
    *   **统一提取引擎**：复用 `ExtractInfoService` 中的 LLM 提取逻辑，确保从文档提取的信息结构与文本输入完全一致。

### 2.1 模块职责划分

#### Backend (Java)
*   `ProjectController`: 新增 `recognizeDocument` 接口，处理文件上传和 Agent 调用。
*   `FileService`: 处理文件存储（本地临时文件或 OSS）。

#### Agent (Python)
*   `DocParser` (Service): 封装阿里云 DocMind SDK，负责将 PDF/Word/Image 转为 Markdown 字符串。
*   `ExtractInfoService`: 
    *   新增 `extract_from_document` 方法。
    *   串联流程：`Document` -> `DocParser` -> `Markdown` -> `LLM (extract_key_factor)` -> `JSON`。

## 3. 详细业务流程

### 3.1 流程图 (Text Description)

1.  **用户动作**: 在 Web 端点击“上传文件识别”，选择 `design_spec.pdf`。
2.  **上传**: Web 调用 Java 接口 `POST /api/project/recognize-doc`。
3.  **存储**: Java 后端接收文件，保存到本地临时目录或上传 OSS，获得 `file_path` 或 `file_url`。
4.  **Agent 调用**: Java 后端调用 Agent 接口 `POST /agent/file/recognize-doc`，Payload: `{ "file_path": "..." }`。
5.  **文档解析 (Agent)**:
    *   Agent 调用 `DocParser` 提交解析任务。
    *   Agent 轮询等待任务完成 (DocMind 处理)。
    *   Agent 获取解析后的 Markdown 文本。
6.  **信息提取 (Agent)**:
    *   Agent 调用 `ExtractInfoService.extract_key_factor(markdown_text)`。
    *   LLM 根据 Prompt 提取项目关键要素 (JSON)。
7.  **返回**: JSON 数据经由 Agent -> Java -> Web。
8.  **展示**: Web 端将 JSON 数据填充到 `wizardForm`，用户可进行校对。

### 3.2 关键逻辑：统一提取 (Unified Extraction)

为了满足“提取内容与文字输入部分一致”的需求，我们**不再依赖** OCR 模型（如 qwen-vl-ocr）直接输出结果，而是采用 **"Parse-then-Extract"** 模式：

*   **输入**: 无论是 PDF、Word 还是 图片。
*   **中间态**: 都通过 DocMind 转换为 **Markdown**。DocMind 能很好地还原图片中的表格结构。
*   **提取器**: 统一使用 `project_info_extract` Prompt。
*   **输出**: 统一的 JSON Schema。

## 4. 模块详细设计

### 4.1 接口定义

#### 4.1.1 Backend API (供前端调用)
*   **URL**: `POST /api/project/recognize-doc`
*   **Content-Type**: `multipart/form-data`
*   **Params**:
    *   `file`: Binary File (PDF/Doc/Image)
*   **Response**:
    ```json
    {
      "code": 200,
      "data": {
        "projectName": "XX高中教学楼",
        "totalArea": 10000,
        "buildingHeight": 50,
        ... // 与 wizardForm 结构一致
      }
    }
    ```

#### 4.1.2 Agent API (供 Backend 调用)
*   **URL**: `POST /agent/extract/doc`
*   **Content-Type**: `application/json`
*   **Body**:
    ```json
    {
      "file_path": "/tmp/uploads/design_spec.pdf", // 本地路径或 URL
      "file_type": "pdf" // 可选
    }
    ```
*   **Response**:
    ```json
    {
      "success": true,
      "data": { ...extracted_json... }
    }
    ```

```python
class ExtractInfoService:
    def extract_from_document(self, file_path: str):
        # 1. 解析文档
        parser = DocParser()
        markdown_content = parser.process_to_markdown(file_path)

        return self.extract_key_factor(markdown_content)
```

## 5. 关键配置项

在 `apps/agent/.env` 中需确保配置：

```ini
ALIBABA_CLOUD_ACCESS_KEY_ID=your_ak
ALIBABA_CLOUD_ACCESS_KEY_SECRET=your_sk
DOCMIND_ENDPOINT=docmind-api.cn-hangzhou.aliyuncs.com
```

## 6. 注意事项

1.  **文件大小限制**: DocMind 通常限制文件大小（如 100MB），后端需做校验。
2.  **解析耗时**: 复杂 PDF 解析可能需要 10-30 秒。前端需展示 Loading 状态（“正在解析文档...”， “正在提取关键信息...”）。
3.  **错误处理**: 如果文档无法解析（加密 PDF、损坏文件），需返回友好错误提示。

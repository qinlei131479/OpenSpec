### 3.3 格式兼容性问题

**问题1: content 格式不适合直接显示**
- 当前 content 包含所有 chunks，以分隔符连接
- 前端需要的是**单章节内容**，而非整个模板

**解决方案:**
- 在模板匹配时，通过章节标题检索，返回**单个 chunk 的内容**
- 新增 API 支持根据 templateId + chapterTitle 获取对应 chunk

**问题2: chapters 与 content 的对应关系**
- chapters 是提取的目录结构
- content 按 `\n---\n\n` 分隔
- 缺少 chapterId → chunkIndex 的直接映射

**解决方案:**
- 方案A: 章节保存时增加 `chunkId` 字段
- 方案B: 通过向量检索直接获取最相关 chunk

## 4. 账号权限设计 (基于 RAGFlow Metadata)

### 4.1 权限模型

```
│                    基于 Metadata 的权限层级                      │

│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4.2 Meta Fields 结构设计

**RAGFlow 使用 `meta_fields` 存储文档元数据 (参考 Python SDK `Document.update` 方法)**

**上传/更新文档时设置的 meta_fields:**

```json
{
  "userId": "user-uuid-or-hash",      // 文档所有者 (必需)
  "isStandard": "false",               // 是否系统标准模板 (必需)
  "templateId": "123",                 // 关联的 DocumentTemplate.id (可选)
  "templateName": "住宅建筑设计说明",   // 模板名称 (可选)
  "tags": "1,2,3",                     // 标签ID列表 (逗号分隔) (可选)
  "fileType": "docx",                  // 文件类型 (可选)
  "uploadTime": "2026-01-27T10:00:00Z" // 上传时间 (可选)
}
```

**说明:**

1. RAGFlow 文档的元数据字段名为 `meta_fields` (dict[str, Any])
2. 布尔值建议转为字符串 `"true"` / `"false"` 以便统一过滤
3. 数组类型需要序列化为字符串，如 `"1,2,3"`
4. 关键字段 `userId` 和 `isStandard` 用于权限控制

### 4.3 检索时的 Meta Fields 过滤

**RAGFlow `retrieve` 方法使用 `metadata_condition` 参数过滤 `meta_fields`**

**RAGFlow 检索参数示例:**

```python
chunks = rag_object.retrieve(
    question=enhanced_query,
    dataset_ids=[personal_template_kb_id],
    page=1,
    page_size=20,
    similarity_threshold=0.5,

    # 核心改动: 使用 metadata_condition 过滤 meta_fields
    metadata_condition={
        "$or": [
            # 条件1: 系统标准模板 (所有人可见)
            {"isStandard": "true"},
            # 条件2: 个人私有模板 (userId 匹配)
            {"userId": user_id, "isStandard": "false"}
        ]
    }
)
```

**metadata_condition 语法说明:**

- 用于过滤文档的 `meta_fields` 字段
- 支持 `$or`, `$and` 逻辑运算符
- 字段名直接对应 meta_fields 中定义的 key

### 4.4 权限控制流程对比

**原方案 (后置过滤):**
```
RAGFlow 检索 (无权限过滤)
  ↓ 返回所有匹配文档
后端 Python 代码遍历结果
  ↓ 逐个查询数据库验证权限
  ↓ 过滤掉无权限的文档
返回过滤后的结果
```

**新方案 (Metadata 前置过滤):**
```
RAGFlow 检索 (带 metadata_filter)
  ↓ RAGFlow 内部完成权限过滤
  ↓ 只返回用户有权访问的文档
直接使用检索结果
```

**优势:**
1. ✅ 检索效率更高 (在 RAGFlow 层面过滤，减少数据传输)
2. ✅ 代码更简洁 (无需后端再次验证权限)
3. ✅ 安全性更好 (权限控制在数据源头)
4. ✅ 扩展性强 (添加新权限维度只需扩展 metadata)

### 4.5 文档 Meta Fields 设置方式

**根据 RAGFlow Python SDK，使用 `Document.update()` 方法设置 `meta_fields`:**

```python
# 方式: 上传文档后，通过 update 方法设置 meta_fields
def set_document_meta_fields(dataset_name: str, doc_id: str, meta_fields: dict):
    """
    设置文档的 meta_fields
    参考 RAGFlow SDK: Document.update(update_message:dict)
    update_message 包含 "meta_fields": dict[str, Any]
    """
    dataset = rag_object.get_dataset(name=dataset_name)
    docs = dataset.list_documents(id=doc_id)
    if docs:
        doc = docs[0]
        # 使用 update 方法设置 meta_fields
        doc.update({
            "meta_fields": meta_fields
        })
        return True
    return False
```

### 5.2 匹配算法详细设计 (基于 Metadata)

```python
async def search_template_chapter(
    user_id: str,
    chapter_title: str,
    template_tags: list[int] = None,
    similarity_threshold: float = 0.5
) -> TemplateMatchResult:
    """
    基于 Metadata 权限过滤的智能匹配

    核心改动:
    - 使用 metadata_filter 在 RAGFlow 层面完成权限过滤
    - 模板信息直接从 chunk.metadata 获取，无需关联数据库

    Args:
        user_id: 用户ID
        chapter_title: 当前章节标题 (e.g., "一、建筑设计说明")
        template_tags: 可选的标签过滤 (e.g., [专业:建筑, 业态:住宅])
        similarity_threshold: 相似度阈值

    Returns:
        TemplateMatchResult: 匹配结果
    """

    # Step 1: 获取模板知识库ID
    kb_ids = get_template_kb_ids()

    # Step 2: 构建检索查询
    enhanced_query = build_enhanced_query(chapter_title)

    # Step 3: 构建 metadata_condition 权限过滤条件 (核心改动)
    metadata_condition = {
        "$or": [
            # 系统标准模板 (所有人可见)
            {"isStandard": "true"},
            # 个人私有模板 (仅所有者可见)
            {"userId": user_id, "isStandard": "false"}
        ]
    }

    # Step 4: 调用 RAGFlow 向量检索 (权限过滤在此完成)
    chunks = rag_object.retrieve(
        question=enhanced_query,
        dataset_ids=kb_ids,
        page=1,
        page_size=20,
        similarity_threshold=similarity_threshold,
        metadata_condition=metadata_condition  # 关键: 过滤 meta_fields
    )

    # Step 5: 构建结果 (无需权限检查，信息从文档 meta_fields 获取)
    # 注意: chunk 关联的文档 meta_fields 需要通过额外查询获取
    scored_results = []
    for chunk in chunks:
        # 获取 chunk 所属文档的 meta_fields
        doc_meta = get_document_meta_fields(chunk.document_id)
        score = calculate_match_score(
            chapter_title=chapter_title,
            chunk_content=chunk.content,
            chunk_tags=doc_meta.get('tags', ''),
            vector_similarity=chunk.similarity,
            target_tags=template_tags
        )
        scored_results.append({
            'templateId': int(doc_meta.get('templateId', 0)),
            'templateName': doc_meta.get('templateName', ''),
            'chunkId': chunk.id,
            'chunkContent': chunk.content,
            'chapterTitle': extract_chapter_title(chunk.content),
            'similarity': score,
            'matchType': 'auto'
        })

    # Step 6: 排序取最优
    scored_results.sort(key=lambda x: x['similarity'], reverse=True)

    if scored_results:
        return TemplateMatchResult(**scored_results[0])

    return None
```

### 5.3 评分函数

```python
def calculate_match_score(
    chapter_title: str,
    chunk_content: str,
    template_tags: list,
    vector_similarity: float,
    target_tags: list = None
) -> float:
    """
    计算综合匹配评分

    评分维度:
    1. 标题相似度 (编辑距离 or 关键词匹配) - 权重 0.3
    2. 向量语义相似度 (RAGFlow 返回) - 权重 0.5
    3. 标签匹配度 - 权重 0.2
    """

    # 1. 标题匹配度
    chunk_first_line = chunk_content.split('\n')[0]
    title_score = calculate_title_similarity(chapter_title, chunk_first_line)

    # 2. 向量相似度 (已由 RAGFlow 计算)
    vector_score = vector_similarity

    # 3. 标签匹配度
    tag_score = 0.0
    if target_tags and template_tags:
        matched_tags = set(target_tags) & set([t.id for t in template_tags])
        tag_score = len(matched_tags) / len(target_tags)

    # 综合评分
    final_score = (
        0.3 * title_score +
        0.5 * vector_score +
        0.2 * tag_score
    )

    return final_score
```

### 8.2 模板匹配服务 (基于 Metadata)

**文件**: `apps/agent/service/tools/template_matcher.py`

```python
import os
import re
import json
from ragflow_sdk import RAGFlow
from typing import List, Optional, Dict, Any
from difflib import SequenceMatcher

RAGFLOW_API_KEY = os.getenv("RAGFLOW_API_KEY")
RAGFLOW_BASE_URL = os.getenv("RAGFLOW_BASE_URL")
PERSONAL_TEMPLATE_KB_NAME = os.getenv("PERSONAL_TEMPLATE_KB_NAME", "personal-template")

class TemplateMatcherService:
    def __init__(self):
        self.rag = RAGFlow(api_key=RAGFLOW_API_KEY, base_url=RAGFLOW_BASE_URL)

    async def search(
        self,
        user_id: str,
        chapter_title: str,
        tag_ids: Optional[List[int]] = None,
        threshold: float = 0.5,
        limit: int = 5
    ) -> Dict[str, Any]:
        """
        基于 Metadata 权限过滤的模板章节检索

        核心改动:
        - 使用 metadata_filter 在 RAGFlow 层面完成权限过滤
        - 无需后置权限验证，检索结果即为用户有权访问的文档
        """

        # 1. 获取模板知识库ID
        kb_ids = self._get_template_kb_ids()

        # 2. 构建增强查询
        enhanced_query = self._build_enhanced_query(chapter_title)

        # 3. 构建权限过滤条件 (核心: 基于 meta_fields)
        metadata_condition = self._build_permission_filter(user_id, tag_ids)

        # 4. 向量检索 (权限过滤在 RAGFlow 层面完成)
        chunks = self.rag.retrieve(
            question=enhanced_query,
            dataset_ids=kb_ids,
            page=1,
            page_size=limit * 2,
            similarity_threshold=threshold,
            metadata_condition=metadata_condition  # 关键: 过滤 meta_fields
        )

        # 5. 构建结果 (无需再次权限检查)
        results = []
        for chunk in chunks:
            # 获取 chunk 所属文档的 meta_fields
            doc_meta = self._get_document_meta_fields(chunk.document_id)

            score = self._calculate_score(
                chapter_title=chapter_title,
                chunk_content=chunk.content,
                vector_similarity=chunk.similarity,
                chunk_tags=doc_meta.get('tags', ''),
                target_tags=tag_ids
            )

            results.append({
                'templateId': int(doc_meta.get('templateId', 0)),
                'templateName': doc_meta.get('templateName', '未知模板'),
                'chunkId': chunk.id,
                'chunkContent': chunk.content,
                'chapterTitle': self._extract_title(chunk.content),
                'similarity': score,
                'matchType': 'auto',
                'documentName': getattr(chunk, 'document_name', '')
            })

        # 6. 排序
        results.sort(key=lambda x: x['similarity'], reverse=True)

        if results:
            return {
                'matched': results[0],
                'alternatives': results[1:limit]
            }
        return {'matched': None, 'alternatives': []}

    def _build_permission_filter(
        self,
        user_id: str,
        tag_ids: Optional[List[int]] = None
    ) -> dict:
        """
        构建基于 meta_fields 的权限过滤条件 (metadata_condition)

        过滤逻辑:
        - 系统标准模板 (isStandard="true") 对所有人可见
        - 个人私有模板 (isStandard="false") 仅对所有者可见
        """
        permission_filter = {
            "$or": [
                # 条件1: 系统标准模板
                {"isStandard": "true"},
                # 条件2: 用户自己的模板
                {
                    "$and": [
                        {"userId": user_id},
                        {"isStandard": "false"}
                    ]
                }
            ]
        }

        # 如果指定了标签过滤，添加标签条件
        # 注意: 由于 tags 存储为逗号分隔字符串，需要特殊处理
        # 这里简化为不在 metadata_condition 中过滤标签，而是在后处理中计算标签匹配度

        return permission_filter

    def _get_document_meta_fields(self, document_id: str) -> dict:
        """获取文档的 meta_fields"""
        try:
            # 通过 document_id 获取文档信息
            # 注意: 这里可能需要缓存优化，避免频繁查询
            datasets = self.rag.list_datasets()
            for ds in datasets:
                docs = ds.list_documents(id=document_id)
                if docs:
                    doc = docs[0]
                    return getattr(doc, 'meta_fields', {}) or {}
        except Exception as e:
            print(f"获取文档 meta_fields 失败: {e}")
        return {}

    def _get_template_kb_ids(self) -> List[str]:
        """获取模板知识库ID"""
        # 从配置获取 personal-template 知识库ID
        kb_id = self._get_kb_id_by_name(PERSONAL_TEMPLATE_KB_NAME)
        return [kb_id] if kb_id else []

    def _get_kb_id_by_name(self, name: str) -> Optional[str]:
        """根据名称获取知识库ID"""
        try:
            datasets = self.rag.list_datasets()
            for ds in datasets:
                if ds.name == name:
                    return ds.id
        except Exception as e:
            print(f"获取知识库失败: {e}")
        return None

    def _build_enhanced_query(self, chapter_title: str) -> str:
        """构建增强查询"""
        # 移除章节编号，提取核心关键词
        clean_title = re.sub(r'^[一二三四五六七八九十\d]+[、\.．]?\s*', '', chapter_title)
        return f"{chapter_title} {clean_title}"

    def _extract_title(self, content: str) -> str:
        """从内容中提取章节标题 (通常是第一行)"""
        if not content:
            return ""
        first_line = content.split('\n')[0].strip()
        # 清理 markdown 格式
        first_line = re.sub(r'^#+\s*', '', first_line)
        return first_line

    def _calculate_score(
        self,
        chapter_title: str,
        chunk_content: str,
        vector_similarity: float,
        chunk_tags: str,
        target_tags: Optional[List[int]]
    ) -> float:
        """
        计算综合匹配评分

        评分维度:
        1. 标题相似度 (权重 0.3)
        2. 向量语义相似度 (权重 0.5)
        3. 标签匹配度 (权重 0.2)
        """
        # 1. 标题匹配度
        first_line = chunk_content.split('\n')[0] if chunk_content else ''
        title_score = self._title_similarity(chapter_title, first_line)

        # 2. 向量相似度 (已由 RAGFlow 计算)
        vector_score = vector_similarity

        # 3. 标签匹配度
        tag_score = 0.0
        if target_tags and chunk_tags:
            # chunk_tags 是逗号分隔的字符串, 如 "1,2,3"
            chunk_tag_ids = set(int(t) for t in chunk_tags.split(',') if t.strip())
            target_tag_set = set(target_tags)
            if target_tag_set:
                matched = chunk_tag_ids & target_tag_set
                tag_score = len(matched) / len(target_tag_set)

        # 综合评分
        final_score = (
            0.3 * title_score +
            0.5 * vector_score +
            0.2 * tag_score
        )

        return round(final_score, 4)

    def _title_similarity(self, title1: str, title2: str) -> float:
        """计算标题相似度 (使用 SequenceMatcher)"""
        if not title1 or not title2:
            return 0.0
        # 清理标题中的编号和空白
        clean1 = re.sub(r'^[一二三四五六七八九十\d]+[、\.．]?\s*', '', title1).strip()
        clean2 = re.sub(r'^[一二三四五六七八九十\d]+[、\.．]?\s*', '', title2).strip()
        return SequenceMatcher(None, clean1, clean2).ratio()
```

```python
def update_document_meta_fields(rag: RAGFlow, dataset_name: str, document_id: str, meta_fields: dict):
    """
    更新 RAGFlow 文档的 meta_fields
    参考 SDK: Document.update(update_message:dict)
    update_message 包含 "meta_fields": dict[str, Any]
    """
    try:
        # 获取 dataset
        datasets = rag.list_datasets(name=dataset_name)
        if not datasets:
            print(f"未找到知识库: {dataset_name}")
            return False

        dataset = datasets[0]

        # 获取文档
        docs = dataset.list_documents(id=document_id)
        if not docs:
            print(f"未找到文档: {document_id}")
            return False

        doc = docs[0]

        # 使用 update 方法设置 meta_fields
        doc.update({
            "meta_fields": meta_fields
        })

        return True

    except Exception as e:
        print(f"更新文档 meta_fields 失败: {e}")
        return False
```

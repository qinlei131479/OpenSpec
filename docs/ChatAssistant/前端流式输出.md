# Generate Node 流式输出修复

## 问题描述

用户反馈 `generate_node` 的输出不是流式的，而是等接口拿到全部输出结果后才显示。

## 问题分析

### 后端流式输出已正确实现

**文件**: [apps/agent/service/workflow/construction_agent.py:L399](apps/agent/service/workflow/construction_agent.py#L399)

```python
streaming_llm = ChatTongyi(model="qwen-max", temperature=0.1, streaming=True)
response = streaming_llm.invoke([HumanMessage(content=full_prompt)], config)
```

- ✅ 后端已使用 `streaming=True`
- ✅ LangGraph 的 `astream_events` 正确捕获流式事件

### API 层流式传输已正确实现

**文件**: [apps/agent/api/workflow_api.py:L257-286](apps/agent/api/workflow_api.py#L257-L286)

```python
# 场景 1: LLM 流式输出 (Streaming Tokens)
if kind == "on_chat_model_stream":
    chunk = data.get("chunk")
    if chunk and chunk.content:
        # 只流式输出用户可见节点的内容
        if node_name in user_visible_nodes:  # user_visible_nodes = {"generate", "general_agent"}
            yield _sse(
                "token",
                {
                    "content": chunk.content,
                    "node": node_name,
                    "timestamp": import_time()
                },
            )
```

- ✅ API 正确发送 SSE `token` 事件
- ✅ `generate` 节点在 `user_visible_nodes` 列表中

### 问题根源：前端事件处理被注释

**文件**: [apps/web/src/components/ChatAssistant.vue:L690-692](apps/web/src/components/ChatAssistant.vue#L690-L692)

**修改前**:
```javascript
} else if (type === 'token') {
    // 流式内容输出（不在侧边栏显示，可选择应用到主编辑区）
    // msg.content += data.content;  // ❌ 被注释了！
```

**问题**:
- `token` 事件虽然被接收，但内容没有追加到 `msg.content`
- 导致用户看不到流式输出效果
- 只能在 `handleGenerate` 的 `fullContent` 中累积，最后一次性显示

## 解决方案

### 修改内容

**文件**: [apps/web/src/components/ChatAssistant.vue:L690-692](apps/web/src/components/ChatAssistant.vue#L690-L692)

**修改后**:
```javascript
} else if (type === 'token') {
    // 流式内容输出 - 实时追加到消息内容中
    msg.content += data.content;  // ✅ 启用流式追加
```

### 工作流程

1. **后端生成** (`generate_node`)
   - 使用 `streaming_llm` 生成内容
   - 每个 token 通过 `on_chat_model_stream` 事件发出

2. **API 传输** (`event_generator`)
   - 捕获 `on_chat_model_stream` 事件
   - 转换为 SSE `token` 事件发送给前端

3. **前端接收** (`workflowChatStream`)
   - 解析 SSE 流
   - 调用 `onEvent` 回调

4. **前端显示** (`handleEvent`)
   - 接收 `token` 事件
   - 实时追加到 `msg.content` ✅
   - Vue 响应式更新 UI

5. **最终应用** (`handleGenerate`)
   - 累积 `fullContent`
   - 生成完成后通过 `emit('content-generated', fullContent)` 应用到编辑器

### 2. 性能优化

```javascript
        throttleTimer = setTimeout(() => {
            scrollToBottom();
            throttleTimer = null;
        }, 100);
    }
}
```

## 总结

问题的根本原因是前端的 `token` 事件处理被注释掉了，导致虽然后端正确发送了流式数据，但前端没有实时显示。

通过取消注释 `msg.content += data.content;` 这一行，即可恢复流式输出功能。

整个流式输出链路：
```
Backend (streaming_llm)
  → LangGraph (on_chat_model_stream)
  → API (SSE token event)
  → Frontend (workflowChatStream)
  → UI (msg.content) ✅
```
